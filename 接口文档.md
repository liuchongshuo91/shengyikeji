# 胜意差旅费用报销系统接口文档

**基础地址：** `http://localhost:8080`

**通用响应格式：**

```json
{
  "code": 0,
  "message": "success",
  "data": ...
}
```

> `code=0` 表示成功，`code=400` 表示业务校验失败。

---

## 1. 报销单管理

### 1.1 查询报销单分页列表

**接口地址：** `POST /api/reimbursements/page`

**接口入参：**

| 字段名 | 类型 | 是否必传 | 备注 | 格式 |
| --- | --- | --- | --- | --- |
| page | Integer | N | 当前页，默认1 | |
| size | Integer | N | 每页大小，默认10 | |
| reimbursementNo | String | N | 报销单号 | |
| reimbursementTitle | String | N | 报销标题 | |
| businessTripReason | String | N | 出差事由 | |
| reimCompanyId | String | N | 费用归属公司ID | |
| reimDepartmentId | String | N | 报销部门ID | |
| reimburserId | String | N | 报销人ID | |
| businessTypeId | String | N | 业务类型ID | |

**入参样例：**

```json
{
  "page": 1,
  "size": 10,
  "reimbursementNo": "",
  "reimbursementTitle": "",
  "businessTripReason": "",
  "reimCompanyId": "",
  "reimDepartmentId": "",
  "reimburserId": "",
  "businessTypeId": ""
```

### 1.2 查询报销单详情

**接口地址：** `GET /api/reimbursements/{id}`

**接口入参：**

| 字段名 | 类型 | 是否必传 | 备注 | 格式 |
| --- | --- | --- | --- | --- |
| id | String | Y | 报销单ID（路径参数） | 32位UUID |

**入参样例：**

```
GET /api/reimbursements/a1b2c3d4e5f60001111222233334444a
```

**接口出参：**

| 字段名 | 类型 | 备注 | 格式 |
| --- | --- | --- | --- |
| code | Integer | 状态码，0=成功，400=失败 | |
| message | String | 提示信息 | |
| data | Reimbursement | 报销单详情（含行程、补助、分摊） | |

data对象在1.1返回字段基础上，额外包含以下嵌套对象：

| 字段名 | 类型 | 备注 |
| --- | --- | --- |
| trips | List | 行程列表 |
| subsidies | List | 补助列表（含日历明细） |
| allocations | List | 费用分摊列表 |

**Trip（行程）字段：**

| 字段名 | 类型 | 备注 | 格式 |
| --- | --- | --- | --- |
| id | String | 主键ID | |
| mainId | String | 关联报销单ID | |
| travelerId | String | 出行人ID | |
| travelerNo | String | 出行人工号 | |
| travelerName | String | 出行人姓名 | |
| fromCityNo | String | 出发城市编号 | |
| fromCityName | String | 出发城市名称 | |
| toCityNo | String | 到达城市编号 | |
| toCityName | String | 到达城市名称 | |
| startDate | String | 出发日期 | yyyy-MM-dd |
| endDate | String | 到达日期 | yyyy-MM-dd |
| tripDescription | String | 行程说明 | |

**Subsidy（补助）字段：**

| 字段名 | 类型 | 备注 | 格式 |
| --- | --- | --- | --- |
| id | String | 主键ID | |
| mainId | String | 关联报销单ID | |
| tripId | String | 关联行程ID | |
| travelerId | String | 出行人ID | |
| travelerName | String | 出行人姓名 | |
| tripDateRange | String | 行程日期范围 | |
| days | Integer | 补助天数 | |
| route | String | 行程路线 | |
| subsidyCity | String | 补助城市 | |
| applyAmount | BigDecimal | 申请金额 | |
| subsidyAmount | BigDecimal | 补助金额 | |
| calendar | List | 日历明细 | |

**CalendarDay（日历明细）字段：**

| 字段名 | 类型 | 备注 | 格式 |
| --- | --- | --- | --- |
| id | String | 主键ID | |
| mainId | String | 关联补助ID | |
| tripDate | String | 行程日期 | yyyy-MM-dd |
| weekName | String | 星期名称 | |
| subsidyCity | String | 补助城市 | |
| mealSelected | Boolean | 是否选中餐费补助 | |
| transportSelected | Boolean | 是否选中交通补助 | |
| phoneSelected | Boolean | 是否选中通讯补助 | |
| mealStandard | BigDecimal | 餐费补助标准 | |
| transportStandard | BigDecimal | 交通补助标准 | 默认40.00 |
| phoneStandard | BigDecimal | 通讯补助标准 | 默认40.00 |
| mealAmount | BigDecimal | 餐费补助金额 | |
| transportAmount | BigDecimal | 交通补助金额 | |
| phoneAmount | BigDecimal | 通讯补助金额 | |

**Allocation（费用分摊）字段：**

| 字段名 | 类型 | 备注 | 格式 |
| --- | --- | --- | --- |
| id | String | 主键ID | |

| reimCompanyNo | String | 费用归属公司编号 | |
| reimCompanyName | String | 费用归属公司名称 | |
| projectId | String | 项目ID | |
| projectNo | String | 项目编号 | |
| projectName | String | 项目名称 | |
| allocationRatio | BigDecimal | 分摊比例 | 合计必须为1（100%） |
| allocationAmount | BigDecimal | 分摊金额 | 合计必须等于补助总金额 |

### 1.3 保存报销单（草稿）

**接口地址：** `POST /api/reimbursements/save`

**接口入参：**

| 字段名 | 类型 | 是否必传 | 备注 | 格式 |
| --- | --- | --- | --- | --- |
| id | String | N | 报销单ID，新增不传，编辑必传 | 32位UUID |
| reimbursementTitle | String | N | 报销标题 | |
| reimburserId | String | N | 报销人ID | |
| reimburserNo | String | N | 报销人工号 | |
| reimburserName | String | N | 报销人姓名 | |
| reimDepartmentId | String | N | 报销部门ID | |
| reimDepartmentNo | String | N | 报销部门编号 | |
| reimDepartmentName | String | N | 报销部门名称 | |
| reimCompanyId | String | N | 费用归属公司ID | |
| reimCompanyNo | String | N | 费用归属公司编号 | |
| reimCompanyName | String | N | 费用归属公司名称 | |
| businessTypeId | String | N | 业务类型ID | |
| businessTypeNo | String | N | 业务类型编号 | |
| businessTypeName | String | N | 业务类型名称 | |
| businessTripReason | String | N | 出差事由 | |
| documentType | String | N | 单据类型，不传默认"日常报销单" | |
| remarks | String | N | 备注 | |
| trips | List | N | 行程列表 | Trip对象，字段同1.2 |
| subsidies | List | N | 补助列表 | Subsidy对象，字段同1.2 |

### 1.5 作废报销单

**接口地址：** `POST /api/reimbursements/{id}/void`

**接口入参：**

| 字段名 | 类型 | 是否必传 | 备注 | 格式 |
| --- | --- | --- | --- | --- |
| id | String | Y | 报销单ID（路径参数） | 32位UUID |

**入参样例：**

```
POST /api/reimbursements/a1b2c3d4e5f60001111222233334444a/void
```

**接口出参（成功）：**

```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

**接口出参（失败 - 单据不存在）：**

```json
{
  "code": 400,
  "message": "单据不存在",
  "data": null
}
```

**接口出参（失败 - 已作废）：**

```json
{
  "code": 400,
  "message": "单据已作废，不可重复作废",
  "data": null
}
```

### 附录：业务规则说明

**补助标准规则：**

| 城市类型 | 餐费补助标准 | 交通补助标准 | 通讯补助标准 |
| --- | --- | --- | --- |
| 一线城市（type=1） | 100.00/天 | 40.00/天 | 40.00/天 |

| 其他城市（type=3） | 50.00/天 | 40.00/天 | 40.00/天 |

| 状态值 | 状态名称 | 说明 |
| --- | --- | --- |
| 0 | 草稿 | 可编辑、可提交、可作废 |

| 2 | 已作废 | 不可编辑、不可重复作废 |
