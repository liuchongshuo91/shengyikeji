# 胜意差旅费用报销系统 - 数据库表结构文档

**数据库类型：** MySQL

---

## 表关系总览

```
fk_reim_main (报销单主表)
  ├── fk_reim_itinerary (行程表)        通过 main_id 关联
  ├── fk_reim_subsidy (补助表)          通过 main_id 关联
  │     └── fk_subsidy_calendar (日历明细表)  通过 main_id 关联补助表id
  └── allocations_json (费用分摊)        以JSON形式存在主表中
```

## 6. 补充说明

### 6.1 主键生成策略

所有表主键均使用 UUID 去横线生成，长度32位。由 Java 端 `UUID.randomUUID().toString().replace("-", "")` 生成。

### 6.2 费用分摊存储方式

费用分摊（Allocation）以 JSON 数组形式存储在主表的 `allocations_json` 字段中，使用 MyBatis-Plus 的 `JacksonTypeHandler` 自动序列化/反序列化。

**JSON结构示例：**

```json
[
  {
    "id": "alloc01a1b2c3d4e5f6a7b8c9d0e1f2",
    "reimCompanyId": "1C54557F1782E000",
    "reimCompanyNo": "",
    "reimCompanyName": "胜意科技北京分公司",
    "projectId": "",
    "projectNo": "",
    "projectName": "",
    "allocationRatio": 1.00,
    "allocationAmount": 420.00
  }
]
```

### 6.3 报销项标记（is_reimbursed）

日历明细表中的 `is_reimbursed` 字段使用逗号分隔的字符串标记当日选中的报销项：

| 值 | 含义 |
| --- | --- |
| meal | 餐费补助 |
| transport | 交通补助 |
| phone | 通讯补助 |

示例：`"meal,transport,phone"` 表示三项都选中；`"meal,phone"` 表示只选中餐费和通讯。

### 6.4 补助标准规则

餐费补助标准根据城市类型决定，存储在日历明细表的 `standard_meal_expenses_amount` 字段中：

| 城市类型 | 一线城市 | 二线城市 | 其他城市 |
| --- | --- | --- | --- |
| type | 1 | 2 | 3 |
| 餐费标准 | 100.00/天 | 80.00/天 | 50.00/天 |
| 交通标准 | 40.00/天 | 40.00/天 | 40.00/天 |
| 通讯标准 | 40.00/天 | 40.00/天 | 40.00/天 |
